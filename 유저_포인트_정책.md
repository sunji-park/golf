# 유저 포인트 정책

**문서 목적**: 유저가 서비스에 기여하는 행위를 **포인트로 보상**하고, 포인트를 **안전하고 공정하게 운용**하기 위한 공식 정책.
**적용 범위**: 유저 포인트의 적립 · 사용 · 등급 · 남용 방지 · 소멸
**관리**: 본 정책은 분기별 리뷰. 변경은 데이터 거버넌스 + 마케팅 + 재무팀 합동 회의 승인 필수.

---

## 1. 개요

포인트는 유저가 **평점·리뷰·사진·친구 초대·라운딩 인증** 등으로 서비스에 기여할 때 적립되는 리워드 단위입니다. 포인트는 **현금성 자산이 아니며**, 서비스 내 혜택(할인·교환·등급 혜택)으로만 사용할 수 있습니다.

- **단위**: 1 P (= 현금 환산 참조값 10 KRW, 재무 내부 가치 평가용)
- **저장 위치**: `user_points` 테이블 (ledger 방식, 모든 적립·차감 기록 보존)
- **원칙**: 유저 기여를 보상하되, **Ballot stuffing·Sybil 공격·무임승차**를 차단

---

## 2. 포인트 적립 규칙

### 2.1 기본 적립 표

| 행위 | 기본 포인트 | 일일 한도 | 품질 조건 |
|------|-------------|-----------|-----------|
| **평점 입력 (L1 라운드 인증 유저)** | 300 P | 10회 | 5개 카테고리 모두 입력 |
| **평점 입력 (L2 회원 자평)** | 100 P | 5회 | 5개 카테고리 모두 입력 |
| **한줄평 작성 (20자 이상)** | +50 P | — | 해당 평가에 귀속 |
| **사진 업로드 (1~10장)** | +20 P/장 | 최대 200 P/평가 | 저작권 본인 소유 |
| **평균타수(스코어) 입력** | +30 P | 평가당 1회 | 60~150 범위, 핸디 구간 함께 기록 |
| **공인핸디캡 인증 후 스코어 입력** | +80 P | 평가당 1회 | 공인핸디캡 등록 + 스코어 |
| **라운딩 인증 업로드 (스코어카드·영수증)** | 500 P | 5회 | 심사 통과 필수 |
| **신규 가입 완료** | 2,000 P | 1회 (평생) | 휴대폰 본인 인증 완료 |
| **친구 초대 성공** | 1,000 P | 10명/월 | 피초대자 L1 첫 평가까지 완료 |
| **피초대자 (가입자)** | 500 P | 1회 | 중복 가입 불가 |
| **프로필 완성 (홈코스·핸디캡 등)** | 200 P | 1회 | 최소 5개 항목 작성 |
| **출석 (일 1회)** | 10 P | 1회/일 | 자동 지급 |
| **연속 출석 7일** | 100 P 보너스 | 주 1회 | 월요일 오전 리셋 |

### 2.2 프리미엄 가산 규칙 (Trust Score 연동)

평점 입력 시 적립 포인트는 **Trust Score 가산** 과 함께 배수 적용됩니다.

```
실제적립 = 기본포인트 × min(1.5, 0.8 + Trust × 0.5)
```

예) Trust 1.0 유저의 L1 평점 → `300 × (0.8 + 1.0×0.5) = 300 × 1.3 = 390 P`
예) Trust 0.3 유저의 L1 평점 → `300 × (0.8 + 0.3×0.5) = 300 × 0.95 = 285 P`
(클램프: 최소 0.95배 ~ 최대 1.5배)

### 2.3 이벤트·프로모션 가산

- **신규 개장 코스 리뷰 TOP 10**: 추가 +500 P
- **계절 특별 리뷰(봄/여름/가을/겨울 시즌 태그 인증)**: 분기당 1회 +200 P
- **파트너 골프장 인증**: 골프장이 지정한 캠페인 참여 시 변동 지급 (마케팅팀 승인)

## 3. 유저 등급 시스템

**누적 적립 포인트(소멸분 제외 기준)** 로 등급이 결정됩니다.

| 등급 | 누적 포인트 | 주요 혜택 |
|------|-------------|-----------|
| **Bronze** | 0 ~ 4,999 P | 기본 기능, 광고 포함 |
| **Silver** | 5,000 ~ 19,999 P | 광고 감소, 주간 푸시 리포트, 쿠폰 월 1장 |
| **Gold** | 20,000 ~ 99,999 P | 광고 제거, 코스 상세 데이터 공개, 파트너 골프장 5% 할인 쿠폰 월 2장 |
| **Platinum** | 100,000 P 이상 | Gold 전체 + 프리미엄 분석 리포트, 우선 예약 대기열, 파트너 골프장 10% 할인 쿠폰 월 3장 |

### 3.1 등급 산정 기준
- **재계산 주기**: 매일 03:00 KST 배치
- **등급 유지 기간**: 해당 등급 획득 후 **12개월간 유지** — 12개월 내 다음 등급 미달성 시 직전 등급으로 1단계 강등
- **강등 최소 보장**: Bronze 이상은 **가입 후 6개월 동안 무강등** (온보딩 보장)

### 3.2 등급별 가중치 보너스
상위 등급은 평점 적립 시 추가 배수를 받습니다.

| 등급 | 포인트 적립 배수 |
|------|-----------------|
| Bronze | 1.00× |
| Silver | 1.05× |
| Gold | 1.10× |
| Platinum | 1.20× |

§2.2 Trust 배수와 **복합 적용** (최종 = Trust 배수 × 등급 배수, 클램프 최대 2.0×).

## 4. 포인트 사용 (Redemption)

### 4.1 사용 가능 항목

| 항목 | 필요 포인트 | 비고 |
|------|-------------|------|
| 파트너 골프장 그린피 할인 쿠폰 (5,000 KRW 상당) | 5,000 P | 월 사용 한도 등급별 차등 |
| 파트너 골프장 그린피 할인 쿠폰 (10,000 KRW 상당) | 10,000 P | 동일 |
| 프리미엄 코스 분석 리포트 (월 1회) | 3,000 P | Gold 이상 무료 |
| 제휴 상품몰 교환 (골프공·마커 등) | 변동 | 재고 연동 |
| 우선 예약 대기열 1회권 | 2,000 P | 인기 코스 한정 |
| 광고 제거 (30일) | 1,500 P | Gold 이상 무료 |

### 4.2 사용 규칙
- **최소 사용 단위**: 500 P
- **유효 기간**: 쿠폰 발행일로부터 **60일**
- **양도 금지**: 타 유저 양도·매매 금지 — 적발 시 전량 회수 및 계정 정지
- **현금 환급 불가**: 법적 예외(서비스 종료·오입금) 외에는 현금 환급 불가

## 5. 남용 방지 (Anti-Abuse)

### 5.1 자동 차단 규칙

| 패턴 | 조치 |
|------|------|
| 동일 IP/디바이스에서 **신규 가입 3건 이상 / 24시간** | 신규 계정 포인트 지급 보류 · 검토 |
| 친구 초대 후 피초대자가 **14일 이내 활동 없음** | 초대자 1,000 P 회수 |
| 허위 라운딩 인증 적발 | 관련 포인트 전량 회수 + Trust Score -0.5 |
| 가입 후 **광고 포인트만 긁고 탈퇴** 패턴 | 재가입 시 포인트 지급 제외 대상 (블랙리스트) |
| 봇 행위 의심 (CAPTCHA 실패 반복) | 포인트 적립 중단 · 운영팀 검토 |

### 5.2 한도 관리
- **일일 최대 적립 한도**: 유저당 **5,000 P/일** (평점·리뷰 활동 기준)
- **월간 최대 적립 한도**: 유저당 **100,000 P/월**
- 한도 초과분은 **다음 날/다음 달로 이월되지 않음** (소멸)

### 5.3 Ledger 감사
- 모든 포인트 적립·차감은 `point_ledger` 테이블에 기록
- 주 1회 (매주 월요일) **자동 이상 적립 스캔** 실행
- 분기 1회 재무팀이 감사

---

## 6. 유효 기간 (Expiration)

- **기본 유효 기간**: 포인트 적립일로부터 **12개월**
- 유저가 기간 내 **추가 적립** 하면 **가장 오래된 포인트부터 소멸 연장** (FIFO 방식)
- 소멸 30일 전 · 7일 전 **2회 알림** (앱 푸시 + 이메일)
- 소멸분은 `point_expired` 테이블에 기록, 감사 목적 보존 5년

---

## 7. 회수 정책 (Clawback)

다음 상황에서 포인트는 **회수**됩니다.

| 사유 | 회수 범위 |
|------|-----------|
| 허위 평점·리뷰로 적립한 포인트 | 해당 건 전량 |
| 친구 초대 허위 (피초대자 활동 없음) | 초대자·피초대자 관련 포인트 전량 |
| 계정 정지 (약관 위반) | 잔여 포인트 전량 소멸 (복구 불가) |
| 시스템 오류로 과다 지급 | 초과분만 회수 (유저 알림 후 7일 경과 적용) |
| 결제 환불 (쿠폰 사용 후 환불) | 사용 포인트 재충전, 쿠폰 무효 |

회수 시 유저에게 **사전 알림 72시간** 을 원칙으로 합니다 (긴급 사안 제외).

---

## 8. 회계·세무 고려사항

- 포인트는 **미인식 부채** 로 재무제표에 반영 — 소멸 시점에 수익 인식
- 포인트 지급을 통해 얻은 **부가적 판촉 효과** 는 마케팅 비용으로 계상
- 골프장 파트너와의 쿠폰 사용 시 **정산 단가** 는 파트너 계약서에 명시
- 연간 3,000만 원 이상 포인트 활동 유저 감사 기록 필요 (국세청 요구 시 제출)

※ 본 섹션은 재무팀 최종 검토 후 세부 회계처리 기준을 추가한다.

## 10. 유저 커뮤니케이션 원칙

- 포인트 지급·차감·소멸은 **유저 앱 내 알림** 으로 즉시 통지
- 포인트 관련 **문의 대응 SLA**: 평일 24시간 이내
- 정책 변경 시 **30일 사전 고지** (앱 내 공지 + 이메일)
- 포인트 내역 **3개월치** 를 언제든 조회 가능 (PDF 다운로드 지원)

## 부록 A — 포인트 Ledger 스키마

```sql
CREATE TABLE point_ledger (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL,
  event_type VARCHAR(40) NOT NULL,   -- 'earn_rating', 'earn_invite', 'spend_coupon', ...
  delta INT NOT NULL,                -- 양수=적립, 음수=차감
  balance_after INT NOT NULL,        -- 트랜잭션 후 잔액
  reference_id BIGINT,               -- 관련 평가/쿠폰/초대 ID
  trust_mult DECIMAL(3,2),           -- 적용된 Trust 배수
  tier_mult DECIMAL(3,2),            -- 적용된 등급 배수
  created_at TIMESTAMPTZ DEFAULT NOW(),
  metadata JSONB                     -- IP·디바이스·캠페인 등
);

CREATE INDEX ON point_ledger (user_id, created_at DESC);
CREATE INDEX ON point_ledger (event_type);
```

## 부록 B — 평점 알고리즘과의 연계

본 포인트 정책은 [**유저 평점 반영 알고리즘 정책**](./유저평점_반영_알고리즘_정책.md)과 **밀접하게 연계**됩니다.

- 평점 **티어 (L1/L2/L3)** 는 포인트 기본 지급액을 결정 (§2.1)
- 평점 **Trust Score** 는 포인트 배수에 반영 (§2.2)
- 평점 **격리·차단** 결정 시 관련 포인트는 자동 회수 (§5.1, §7)
- 평점 **유저 등급** 이 곧 포인트 등급에 반영되며 서로 상호 강화

두 정책의 변경은 **상호 참조 후 동시 개정**을 원칙으로 한다.
