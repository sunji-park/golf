# GolfNow 런칭 전 코스 평점 확보 전략 (저예산 · 크롤링 중심)

**대상 데이터**: `golfcourse_database.xlsx`의 평점·성적 컬럼
- J열 티박스 / K열 페어웨이 / L열 그린 / M열 페이스 / N열 캐디 (각 1~5점)
- O열 종합평점 = `ROUND(AVERAGE(J:N), 2)` 자동계산
- **P열 평균타수** (18홀 기준 아마추어 평균 스코어, **신규**)
- Q열 슬로프 / R열 코스레이팅 / S열 난이도 (코스 공식값)

**규모**: 국내 510개 · 일본 140개 = **총 650개 코스**
**런칭 목표**: **650개 전 코스**의 J~N열을 런칭일까지 채운다 (커버리지 100%)
**예산 목표**: **총 30만 원 이하** (가능하면 10만 원대로)

## 3. 리뷰 → 5개 카테고리 점수 + 평균타수 변환 파이프라인

### 3.1 핵심 아이디어
수집한 블로그·리뷰 텍스트를 LLM에게 **JSON 출력**으로 강제해서 **5개 카테고리 점수 + 핸디캡 구간별 평균타수**를 뽑아낸다.

### 3.2 프롬프트 템플릿
```
당신은 골프 코스 평가 전문가입니다. 다음 후기를 읽고 평점과 평균 타수를 추출하세요.
없거나 언급 안 된 항목은 null로 두세요.

카테고리 정의 (각 1~5점):
- 티박스: 티잉 그라운드 관리, 디벗·잔디 품질
- 페어웨이: 페어웨이 잔디 밀도·라이 상태
- 그린: 그린 스피드·굴곡·관리
- 페이스: 라운딩 진행 속도 (홀 간 대기)
- 캐디: 캐디 서비스 친절도·전문성 (노캐디면 null)

평균타수 (18홀 기준, 아마추어 기준):
- beginner: 핸디캡 30+ 예상 스코어
- casual: 핸디캡 20~30 생활골퍼
- intermediate: 핸디캡 10~20 중급자
- advanced: 핸디캡 5~10 상급자
- pro_am: 핸디캡 0~5 최상위
- overall_avg: 일반 성인 아마추어 평균 (casual과 intermediate 사이)

후기:
"""
{review_text}
"""

코스 정보 참고: 파 {par}, 슬로프 {slope}, 코스레이팅 {course_rating}

출력 (JSON):
{
  "teebox": ?, "fairway": ?, "green": ?, "pace": ?, "caddy": ?,
  "avg_score_beginner": ?, "avg_score_casual": ?, "avg_score_intermediate": ?,
  "avg_score_advanced": ?, "avg_score_pro_am": ?, "overall_avg": ?,
  "confidence": 0~1
}
```

### 3.3 집계 규칙
코스당 수집한 리뷰 N개를 LLM에 돌려 N개 JSON을 받은 뒤:

```
코스 카테고리 점수 = 가중평균(점수_i × confidence_i × recency_i)
```

- `recency_i` = 최근 1년 이내 리뷰 1.0, 1~2년 0.7, 2~3년 0.4, 3년+ 0.2
- null 카테고리는 평균에서 자동 제외
- 최종적으로 `round(X, 1)` 로 1자리 소수 점수 (UX와 일관)

### 3.4 LLM 비용 추정
- 평균 리뷰 300자 한국어 ≈ 200 토큰 입력, JSON 50 토큰 출력
- 650 코스 × 평균 50개 리뷰 = **32,500건**
- GPT-4o-mini 기준: $0.15/1M 입력 + $0.60/1M 출력 → **약 $5 (~7,000원)**
- Claude Haiku도 유사
- 여유 있게 잡아도 **3~5만 원** 이면 전체 650개 코스의 리뷰 처리 완료

## 5. 실행 플로우 (5단계 자동화)

```
[Step 1] 엑셀에서 650개 코스명·지역 추출
   └─> crawlers/courses.json 생성

[Step 2] 각 소스별 크롤러 병렬 실행
   ├─> 네이버 플레이스 크롤러 (Selenium)
   ├─> 네이버 블로그 API 크롤러 (검색 API)
   ├─> 카카오맵 크롤러 (Playwright)
   ├─> 구글 리뷰 크롤러 (Places API 또는 Selenium)
   ├─> 골프 버티컬 크롤러 (bs4)
   └─> 일본 Rakuten GORA · GDO 크롤러

[Step 3] 수집된 raw 리뷰를 코스별로 매칭
   └─> crawled/{course_id}.jsonl (중복 제거 후)

[Step 4] LLM 카테고리 추출 파이프라인
   └─> 병렬 API 호출 (asyncio) → scored/{course_id}.json

[Step 5] 가중평균 집계 후 엑셀 J~N열에 write-back
   └─> golfcourse_database.xlsx 업데이트
```

### 예상 소요시간
- Step 1: 수분
- Step 2: **2~6시간** (병렬 크롤러 5개 기준)
- Step 3: 수분
- Step 4: **1~2시간** (LLM 병렬 호출)
- Step 5: 수분

**총 반나절** 이면 650개 코스 J~N열이 모두 채워진다.

## 9. 산출물 매핑 (엑셀 직접 연결)

| 엑셀 컬럼 | 채우는 방식 |
|-----------|-------------|
| **J 티박스** | LLM 추출 "teebox" 가중평균, 1자리 반올림 |
| **K 페어웨이** | LLM 추출 "fairway" 가중평균 |
| **L 그린** | LLM 추출 "green" 가중평균 |
| **M 페이스** | LLM 추출 "pace" 가중평균 |
| **N 캐디** | LLM 추출 "caddy" 가중평균 (노캐디 코스는 빈 값) |
| **O 종합평점** | 엑셀 수식 자동 계산 `ROUND(AVERAGE(J:N), 2)` |
| **P 평균타수 (신규)** | **§9.1 참조** — 리뷰 스코어 인용 + 코스레이팅/슬로프 계산 |
| **Q 슬로프** | 공식 데이터(골프장 홈페이지) 별도 크롤러로 수집 |
| **R 코스레이팅** | 동일 |
| **S 난이도** | 슬로프/코스레이팅 기반 자동 분류 로직 |
| **비고 (마지막 열)** | 리뷰 수, 소스, confidence, **핸디캡 구간별 평균타수** 등 메타 정보 |

### 9.1 평균타수 수집 파이프라인 (별도 방법론)

평균타수는 리뷰 텍스트에서 직접 추출하기 어려우므로 **3단계 혼합 방식**으로 채운다.

**방법 A — 리뷰 텍스트 직접 인용 (1순위)**
- 네이버 블로그·김캐디·클리앙 골프갤에서 _"오늘 XX치고 왔습니다"_, _"핸디 X로 XX 침"_ 같은 스코어 인용 패턴 추출
- 코스당 최소 10건 이상 확보되면 **중앙값** 사용
- 정규식: `(?:스코어|성적|결과|쳤|친|파)\s*(\d{2,3})`, `(?:핸디|핸디캡)\s*(\d+)`

**방법 B — 핸디캡 × 코스레이팅 공식 (2순위·표준)**
- USGA·R&A 표준 공식으로 계산:
```
avg_score = par + (course_rating - par) + (slope / 113) × handicap_median
```
- 구간별 핸디캡 중앙값:
  - beginner: 32 / casual: 24 / intermediate: 15 / advanced: 7 / pro_am: 2
- 예) Hirono (par 70, CR 74.3, slope 143) → intermediate = 70 + 4.3 + (143/113)×15 ≈ 93

**방법 C — 토너먼트 개최지 보정 (3순위·공신력)**
- KPGA·JLPGA·PGA 개최 코스는 공식 아마추어 토너먼트 스코어 분포 데이터 참고
- 예: CJ컵 개최 클럽나인브릿지 → 아마추어 중급 평균 96~100

### 9.2 평균타수 5개 구간 저장

엑셀 P열(평균타수)에는 **overall_avg 1개 값**을 표시. 세부 5구간 스코어는 **비고 열**에 저장:

```
평균타수(핸디별): 초보 110/ 생활 102/ 중급 95/ 상급 82/ 프로급 73 · 난이도등급 상급
```

런칭 UI에서 유저는 본인 핸디캡을 선택하면 해당 구간 스코어를 보게 되며, 평균타수와 본인 목표 스코어를 비교 가능.

### 9.3 평균타수 크롤링 소스 우선순위

| 순위 | 소스 | 특징 |
|------|------|------|
| 1 | 네이버 블로그 "○○cc 라운딩 후기" | 스코어 직접 인용 다수 |
| 2 | 김캐디·골프존카운티·XGolf 리뷰 | 핸디캡별 피드백 |
| 3 | 클리앙·딜바다 골프포럼 | 솔직한 스코어 공유 |
| 4 | 공식 슬로프·코스레이팅 (Q·R열) | 공식 수학 보정 |
| 5 | 토너먼트 스코어 데이터 (KPGA 기록) | 개최 코스 상세 |
| 6 | Rakuten GORA · GDO 일본 평균 스코어 섹션 | 일본 코스 공식 데이터 |

## 10. 런칭 후 자연 전환

런칭 직후 유저가 평점을 입력하기 시작하면, **크롤링 시드 점수는 prior**로 전환:

```
R_final = (seed_n × R_seed + user_n × R_user) / (seed_n + user_n)
```

- 크롤링 시드: `seed_n = 30` (신뢰도)
- 유저 평가가 30건 쌓이면 **유저 평점 비중 50%**, 100건 쌓이면 **80%**
- 6개월 후 크롤링 시드는 자동 소멸 (유저 데이터로 완전 대체)

이렇게 하면 **런칭일 커버리지 100%** 를 유지하면서도 시간이 지나면 자연스럽게 진짜 유저 평점으로 수렴한다.
