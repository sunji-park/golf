# 유저 평점 반영 알고리즘 정책

**문서 목적**: 유저가 입력한 코스 평점·평균타수를 **어떻게 받고 / 어떻게 집계하고 / 어떻게 공개할지**를 정의하는 공식 운영 정책.
**적용 대상**: `golfcourse_database.xlsx`의 5개 카테고리(티박스·페어웨이·그린·페이스·캐디) + 종합평점 + **평균타수**
**관리**: 본 정책은 분기별 리뷰. 변경은 데이터 거버넌스 회의 승인 필수.

---

## 1. 설계 원칙

1. **Ballot stuffing 방지** — 한 명이 수백 개 평가해 순위 뒤집지 못하게 (단, **정상적인 반복 라운딩은 허용**)
2. **최근성 반영** — 5년 전 평가와 어제 평가는 다르게 가중
3. **라운딩 인증 가중** — 스코어카드·영수증 등 외부 인증 유저 점수가 더 무겁게
4. **카테고리별 독립 관리** — 그린만 나쁜 코스와 캐디만 나쁜 코스를 섞지 않기
5. **투명성** — 알고리즘 수식은 공개 가능한 수준으로 문서화
6. **진성 유저 친화** — 정책은 **완화를 기본**으로, 이상 패턴은 통계적으로 걸러내는 구조

---

## 2. 평점 입력 UX 규칙

### 2.1 입력 권한 (Eligibility)

유저는 **다음 중 하나 충족 시** 해당 코스 평점 입력 가능 (3단계):

| 레벨 | 조건 | 가중치 w_user |
|------|------|--------------|
| **L1 — 라운드 인증 유저** | 스코어카드 / 결제 영수증 / 출입증 / 캐디피 영수증 중 1건 이상 업로드 심사 통과 | **1.0** |
| **L2 — 회원 자평** | 가입 후 7일 경과 & 본인 인증(휴대폰/이메일) 완료 | **0.5** |
| **L3 — 게스트** | 이메일만 인증, 라운딩 정보 없음 | **0.0 (표시만, 집계 제외)** |

### 2.2 입력 주기 제한

**원칙: 느슨하게 열고, 이상 패턴은 §5 통계 필터로 걸러낸다.**

| 항목 | 기준 |
|------|------|
| 동일 코스 재평가 주기 | **제한 없음** — 이전 평가는 과거 시점 데이터로 유지, 최신 평가도 신규 데이터로 독립 집계 |
| 동일 IP/디바이스 + 동일 코스 | **제한 없음** — 같은 계정은 **동일 일자 + 동일 코스 1회**만 허용 (최신 건 UPSERT) |
| 하루 최대 평가 코스 수 | **15개** (대회 참가·연속 라운딩·언론 취재 등 예외 케이스 수용) |
| 같은 코스 반복 라운딩 | **정상 허용** (예: 이틀 연속 · 주말마다 같은 홈코스 라운딩) |

#### 2.2.1 반복 라운딩 예시 처리

- **케이스 A**: 토·일 이틀 연속 같은 코스 라운딩 후 평점 입력
  → 이틀 각각 **독립 평가로 저장**. 날짜 태그로 구분. 집계 시 둘 다 유효 표본.

- **케이스 B**: 주간 4회(월·수·금·일) 동일 홈코스 라운딩
  → 4건 모두 유효. 단 **Trust Score의 "다수 의견 편차"** 가 낮아지면 자연스럽게 가중치 자기조정.

- **케이스 C**: 같은 날짜에 동일 코스 2건 입력 (시스템 실수·고의)
  → **가장 최근 제출 건만 유효**. UI 상 "오늘 제출 건이 있어 갱신됩니다" 경고.

#### 2.2.2 이상 패턴 감지 (Guard Rail)

단순 Rate limit이 아니라 **이상 패턴 감지 기반**:

- 한 계정 **최근 7일 평가 수가 평균 유저 대비 상위 0.5% 이상** → 운영팀 검토 큐
- **같은 IP 클러스터에서 5명 이상 동일 코스에 동일 시간대 평가** → Sybil 의심 플래그 (§4.2)
- 신규 가입 24시간 이내 **동일 방향(전부 5점 또는 전부 1점) 평가 3건 이상** → 일시 보류

이 감지 규칙은 **자동 차단이 아닌 격리·검토** 로 동작 — 진성 유저 체험을 해치지 않는다.

### 2.3 입력 필드
- 필수: 5개 카테고리 각 1~5점 (소수점 없음, 정수만)
- 선택: 한줄평(20자 이상 시 +0.1 가중치), 사진 1~10장 (+0.1), 라운딩 일자
- **평균타수(스코어) 입력**: 선택 항목 — 본인의 18홀 스코어 정수 입력 (60~150 범위) · §2.3.1 참조
- 종합평점은 **자동 계산** (유저 직접 입력 금지 — 조작 여지 차단)
- 반복 라운딩 시 **라운딩 일자 필수 입력** 권장 (편의상 기본값 = 오늘, 수정 가능)

#### 2.3.1 평균타수 입력 UX

유저가 평점 입력 시 본인 **18홀 총 타수**를 함께 기록할 수 있도록 폼에 입력 필드를 제공한다.

| 필드 | 입력 방식 | 검증 규칙 |
|------|-----------|-----------|
| **본인 스코어** | 숫자 입력 (60~150) | 범위 외 입력 불가 |
| **본인 핸디캡 구간** | 5단계 라디오 (beginner/casual/intermediate/advanced/pro_am) | 프로필 값 자동 채움, 수정 가능 |
| **라운딩 일자** | 캘린더 | 기본 오늘, 미래 선택 불가 |
| **공인핸디캡 여부** | 체크박스 | 체크 시 가중치 1.2× |

집계 규칙:
- 입력된 스코어는 `course_scores` 테이블에 저장
- 코스별 **핸디캡 구간별 중앙값**을 집계해서 `avg_score_{구간}` 을 갱신
- **overall_avg** = 구간별 중앙값의 가중평균 (각 구간 유저 수 비례)
- 극단값(코스 par - 5 미만 / par + 60 초과) 자동 격리
- 시간 감쇠 없음 — 코스 난이도는 시간에 따라 크게 변하지 않으므로 전체 기록을 누적 사용

집계 공식:
```
avg_score_구간 = median(scores_구간)
overall_avg = Σ(count_구간 × avg_score_구간) / Σ count_구간
```

유저에게 노출:
- 본인 핸디캡 구간의 평균타수 + 전체 overall_avg 함께 표시
- "당신 구간(중급) 평균: 95타 / 전체 평균: 98타" 식으로 비교 가능
- n < 5 인 구간은 "표본 부족" 뱃지

## 3. 핵심 집계 공식

### 3.1 단일 코스·단일 카테고리 점수

```
R_final = (Σ w_i × s_i × decay(t_i) + m × R_prior) / (Σ w_i × decay(t_i) + m)
```

| 기호 | 의미 | 값/규칙 |
|------|------|---------|
| s_i | i번째 유저 점수 | 1~5 정수 |
| w_i | 유저 가중치 | 2.1의 L1(1.0) 또는 L2(0.5) + 신뢰도 보정(§4) |
| decay(t_i) | 시간 감쇠 | `exp(-Δt / 730)` (반감기 2년, Δt=일수) |
| m | Bayesian prior 강도 | **20** (신규 코스 안정화) |
| R_prior | 사전 평균 | 해당 지역·세부종류 코스의 전체 평균 |

### 3.2 종합평점
```
종합평점 = ROUND(AVERAGE(티박스, 페어웨이, 그린, 페이스, 캐디), 2)
```
— 엑셀 수식과 정책 일치. 카테고리별 불균형 가중은 향후 별도 검토.

### 3.3 표시 규칙
- n(유효 표본) < 5 → **"신규 · 평가 수집 중"** 뱃지 표시, 숫자 공개 X
- 5 ≤ n < 15 → 점수 공개하되 "± 0.x" 오차 범위 함께 표시
- n ≥ 15 → 점수만 표시

## 4. 신뢰도(Trust) 보정 시스템

각 유저에게 **Trust Score (0.0~1.5)** 를 부여하고 가중치에 곱한다.

### 4.1 Trust Score 가산 요인
| 항목 | 가산 |
|------|------|
| 계정 나이 365일 이상 | +0.15 |
| 누적 L1 라운드 인증 평가 10회 이상 | +0.20 |
| 사진·한줄평 첨부율 50% 이상 | +0.10 |
| 다수 의견과의 편차가 평균 이하 (Z-score<1.0) | +0.15 |
| 관리자 인증 전문가(KPGA/KLPGA/티칭프로) | +0.40 |
| 동일 홈코스 반복 라운딩 인증 (연 12회 이상) | +0.10 |

### 4.2 Trust Score 감산·차단 요인
| 항목 | 감산/제재 |
|------|----------|
| 극단 점수만 입력(1점 또는 5점 비율 ≥ 80%) | −0.30 |
| 라운딩 인증 없는 코스 평가 이력 누적 | −0.20 |
| 욕설·광고·상표권 침해 리뷰 | −0.50, 2회 누적 시 **영구 차단** |
| 동일 IP/디바이스 클러스터 5계정 이상 동시 감지 | −1.0 (Sybil attack 방지) |

최종 적용: `w_i = 기본가중치(L1=1.0 / L2=0.5) × clamp(Trust, 0.1, 1.5)`

---

## 5. 이상치 탐지 (Outlier Rejection)

각 코스별로 새 평가가 들어올 때:

1. 직전 90일 유효 평가의 **중앙값 m** 과 **MAD(중앙절대편차) σ** 계산
2. 신규 평가의 Z-score > **3.0** 이면 격리 → 관리자 검토 큐
3. 동일 코스에서 같은 Trust Score의 **연쇄 극단 점수(5건 이내 같은 방향으로 ±1점 이상 밀림)** → 조작 의심 플래그

격리된 평가는 **집계에서 제외**하되, 삭제하지 않음(감사 로그 유지).

## 6. 반영 주기 (Update Cadence)

| 레이어 | 갱신 주기 | 기술 |
|--------|-----------|------|
| 내부 DB raw 평가 테이블 | 실시간 | INSERT/UPDATE |
| 집계된 공개 평점 | **매일 03:00 KST 배치** | Airflow DAG |
| 카테고리 prior (R_prior) | 월 1회 | Materialized view |
| Trust Score 재계산 | 주 1회(일요일) | Batch |

긴급 수정이 필요한 경우(예: 악성 리뷰 급증) **즉시 재집계 큐**를 수동 트리거 가능.

## 7. 카테고리별 특수 규칙

### 티박스
- 공사·변경 이벤트 플래그가 있으면 해당 이전 평가 **가중치 50% 감소**

### 페어웨이 · 그린
- 계절성 인정: 입력 시 **시즌 태그**(성수기/비수기) 자동 부여. 공개 시 "지난 12개월" 기본, 필터로 계절별 볼 수 있음

### 페이스(진행 속도)
- 평일/주말 시간대 태그와 연동 → 주말 오후 느린 진행은 정상값이므로 **별도 baseline** 적용

### 캐디
- **캐디 개별 식별 지양** — 개인정보·노동인권. 코스 단위 집계만
- "노캐디/셀프" 코스는 **N/A 처리**, 평균에서 자동 제외 (엑셀 가중치 수식 재조정)

## 8. 분쟁 처리 (Dispute Flow)

```
[골프장 이의신청] → 평가 ID·사유 제출
    ↓
[자동검토] 해당 유저 Trust < 0.5 & 내용 확인 가능 → 일시 숨김
    ↓
[수동심사] 운영팀 48시간 내 판단
    ↓
[결과] 유지 | 삭제 | 유저 감산 | 패널티 조정
```

골프장이 특정 평가 **삭제를 강요해 결과를 미화**하려는 시도는 거절. 명백한 허위·모욕에만 개입.

## 9. 공정성 감사 (Fairness Audit)

분기별로 다음 지표를 모니터링:

- 유료 광고 골프장과 일반 골프장 평균 차이 **≤ 0.3점** (확인 실패 시 랭킹 광고 분리)
- 상위 10% / 하위 10% 평점 코스의 **재라운딩 의향 CTR** 이 실제 점수와 상관계수 0.5 이상
- Trust 하위 20% 유저만으로 집계한 결과와 전체 결과의 상관계수 0.7 이상 (극단 유저 영향력 확인)

## 11. 알고리즘 의사코드 (참조 구현)

```python
from math import exp
from statistics import median

def recalc_rating(course_id: str, category: str) -> dict:
    reviews = db.query_valid_reviews(course_id, category)   # L1/L2만, L3·격리 제외
    now = today()
    prior = prior_by_region_type(course_id, category)        # §3.1 R_prior
    M = 20                                                    # §3.1 Bayesian m

    num, denom = 0.0, 0.0
    scores_weighted = []
    for r in reviews:
        delta_days = (now - r.date).days
        decay = exp(-delta_days / 730)                        # §3.1 2년 반감기
        trust = clamp(r.user.trust, 0.1, 1.5)
        # 티어: L1=1.0(라운드 인증) / L2=0.5(회원 자평) / L3=0.0(집계 제외, 사전 필터됨)
        lvl_w = {"L1": 1.0, "L2": 0.5}[r.user.level]          # §2.1
        bonus = (0.1 if len(r.text) >= 20 else 0) + (0.1 if r.photos else 0)
        w = lvl_w * trust + bonus

        num += w * r.score * decay
        denom += w * decay
        scores_weighted.append(r.score)

    # Outlier rejection (§5)
    m, s = median(scores_weighted), mad(scores_weighted)
    reviews_clean = [r for r in reviews if abs(r.score - m) <= 3 * s] if s else reviews

    R = (num + M * prior) / (denom + M) if (denom + M) else prior
    n_effective = len(reviews_clean)
    ci = 1.96 * stdev_weighted(reviews_clean) / max(1, n_effective ** 0.5)

    return {
        "score": round(R, 2),
        "n": n_effective,
        "confidence_interval": round(ci, 2),
        "display": "score" if n_effective >= 15 else ("score_with_ci" if n_effective >= 5 else "collecting"),
    }
```

## 부록 A — 엑셀 스프레드시트 매핑

`golfcourse_database.xlsx`의 평점·평균타수 컬럼은 **집계 결과의 스냅샷**으로만 사용되며, 실시간 원천은 DB. 월 1회 운영팀이 스냅샷을 엑셀에 export한다.

| 시트 컬럼        | 매핑 필드                            |
|------------------|--------------------------------------|
| 티박스(1-5)      | `course_ratings.teebox_score`        |
| 페어웨이(1-5)    | `course_ratings.fairway_score`       |
| 그린(1-5)        | `course_ratings.green_score`         |
| 페이스(1-5)      | `course_ratings.pace_score`          |
| 캐디(1-5)        | `course_ratings.caddy_score`         |
| 종합평점         | `AVERAGE` 수식 유지 (§3.2)           |
| **평균타수**     | `course_avg_scores.overall_avg` (핸디 구간별 중앙값 가중평균) |
| 슬로프 / 코스레이팅 | **유저 평점 아님** — 공식 측정값    |
| 난이도           | 슬로프·코스레이팅 기반 자동 분류     |

### A.1 course_avg_scores 스키마

```sql
CREATE TABLE course_scores (
  id BIGSERIAL PRIMARY KEY,
  course_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  handicap_tier VARCHAR(16),        -- beginner/casual/intermediate/advanced/pro_am
  score INT NOT NULL CHECK (score BETWEEN 60 AND 150),
  verified_handicap BOOLEAN DEFAULT FALSE,
  round_date DATE NOT NULL,
  rating_id BIGINT REFERENCES course_ratings(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

- 핸디 구간별 중앙값은 `course_avg_scores` 머티리얼라이즈드 뷰로 생성
- 매일 03:00 KST 배치로 갱신
